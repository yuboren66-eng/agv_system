## 小车收信息含义

所有的cmd信息包括`type`字段，该字段为`string`。

| type     | Qos  | 含义                | 备注     |
| -------- | ---- | ------------------- | -------- |
| `ORIENT` | 2    | 直角/直行的转向指引 | 测试使用 |
| `ANGLE`  | 2    | 带角度的转向指引    | 暂不开发 |
| `QUERY`  | 1    | 主机查询信息        | 暂不开发 |
| `ACTION` | 2    | 主机行动命令        | 部分使用 |

### 直角/直行的转向指引

小车收到该信号后，如果没有启动则启动，在下个路口按照指引前进。如果在路口没有收到信息，请停车并发送`INFO`。

| 键    | 键备注                        | 值         | 值备注                     |
| ----- | ----------------------------- | ---------- | -------------------------- |
| type  | -                             | ORIENT     | -                          |
| param | 下个路口方向<br />enum string | *STRAIGHT* | 直行                       |
|       |                               | *LEFT*     | 左拐                       |
|       |                               | *RIGHT*    | 右拐                       |
|       |                               | *ARRIVED*  | 到达<br />按照br的要求停车 |
|       |                               | UTURN      | 掉头                       |



### 带角度的转向指引

预期功能和直角/直行的转向指引相似，br和lmx安排，暂不开发。



### 主机查询信息

预期：小车收到信号后，立即返回索要的信息。暂不开发，这里列出设计字段。

| 键    | 键备注                    | 值       | 值备注 |
| ----- | ------------------------- | -------- | ------ |
| type  | -                         | QUERY    | -      |
| param | 查询内容<br />enum string | *STATUS* |        |
|       |                           | *LOG*    | -      |



### 主机行动命令

小车收到信号后，完成对应动作。

| 键    | 键备注                    | 值        | 值备注                             |
| ----- | ------------------------- | --------- | ---------------------------------- |
| type  | -                         | ACTION    | -                                  |
| param | 动作内容<br />enum string | *PAUSE*   | 暂停前进                           |
|       |                           | *PROCESS* | 恢复暂停                           |
|       |                           | UTURN     | 掉头<br />相关逻辑需要和br/lmx确认 |
|       |                           | *REBOOT*  | 重启<br />暂不开发                 |



## 小车发信息含义

所有的cmd信息包括`type`字段，该字段为`string`。

| type       | Qos  | 含义               | 备注     |
| ---------- | ---- | ------------------ | -------- |
| `ARRIVE`   | 2    | 到达路口或终点指引 | 测试使用 |
| `OBSTACLE` | 2    | 遇见障碍的信号     | 测试使用 |
| `REPAIRED` | 2    | 障碍恢复的信号     | 测试使用 |
| `ACK`      | 2    | 回应主机查询信息   | 暂不开发 |
| `POSITION` | 2    | 位置信息           | 测试使用 |



### 到达路口或终点指引

行进中的小车到达路口1（路口1已提前知道怎么走了）后，立刻发送。

| 键    | 键备注 | 值     | 值备注 |
| ----- | ------ | ------ | ------ |
| type  | -      | ARRIVE | -      |
| param | 空     | ""     |        |



###   遇见障碍的信号

行进中的小车遇见障碍，立即停车，立即发送。

| 键    | 键备注                                       | 值                                 | 值备注                                                       |
| ----- | -------------------------------------------- | ---------------------------------- | ------------------------------------------------------------ |
| type  | -                                            | OBSTACLE                           | -                                                            |
| param | 是否已过编码器阈值标志位<br />障碍类型<br /> | 复合\<string\><br />"y/n"+障碍名称 | 总长不应超过8位<br />y-已过阈值；n-未过阈值<br />举例"yapple"，"nbanana" |



### 障碍恢复的信号

碰到障碍修复，立即发送。

| 键    | 键备注 | 值       | 值备注 |
| ----- | ------ | -------- | ------ |
| type  | -      | REPAIRED | -      |
| param | 留空   | ""       | -      |



### 回应主机查询信息

预期：小车收到信号后，立即返回索要的信息。暂不开发。

### 位置信息

摆放到新位置后发送

| 键    | 键备注                  | 值                                            | 值备注                     |
| ----- | ----------------------- | --------------------------------------------- | -------------------------- |
| type  | -                       | POSITION                                      | -                          |
| param | 背对节点+朝向节点<br /> | 拼接\<string\><br />背对节点号+"-"+朝向节点号 | 举例<br />"2-8"<br />"7-1" |
